## Context

The inability to revoke JWTs became a real liability during a security review. We need immediate revocation and the ability to change claims mid-session.

## Decision

Issue opaque session identifiers backed by a Redis session store. Revocation and privilege changes take effect on the next request.

## Consequences

- Instant logout and revocation across all devices.
- The session store is now a hot dependency requiring replication.
- Sessions are no longer self-contained. Supersedes ADR-0005.
