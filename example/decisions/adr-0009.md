## Context

An enterprise contract requires a recovery story that survives the loss of an entire region. We are not yet ready to run writes in two places at once.

## Decision

A second region runs warm with asynchronous database replication. Failover is a deliberate, rehearsed operation rather than automatic.

## Consequences

- Survives a regional outage with bounded data loss.
- The passive region is paid-for capacity sitting mostly idle.
- Active-active was later proposed to use that capacity — see ADR-0018.
