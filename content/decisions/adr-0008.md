---

## Context

Trunk-based development means half-finished work reaches production daily. We need to merge continuously while controlling what users actually see.

## Decision

All runtime toggles resolve through a single flag service with an SDK in each app. Deploying code and releasing a feature become separate, independently reversible actions.

## Consequences

- Safer, more frequent merges and instant kill switches.
- Flags accumulate as debt unless they are actively retired.
- The flag service becomes a request-path dependency that must be highly available.
