## Context

Global customers see avoidable latency on catalogue and metadata reads that change infrequently but are requested constantly.

## Decision

Serve cacheable reads from an edge cache keyed by tenant and resource, invalidated by an event on the corresponding write. Authenticated mutations always hit origin.

## Consequences

- Dramatically lower read latency for distant clients.
- Correct invalidation is the hard part and the main risk.
- Cache keys must be tenant-scoped to avoid any cross-tenant leakage.
