## Context

We want horizontal scaling without sticky sessions or a shared session store, and JWTs promise exactly that — the token carries its own claims.

## Decision

Authenticate with short-lived signed JWTs kept by the client; the API validates the signature on each request and holds no session state.

## Consequences

- No session store to operate in the early days.
- Revocation is genuinely hard — a leaked token is valid until it expires.
- Claims changes can't take effect mid-session. This approach was retired in ADR-0012.
