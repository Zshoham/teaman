## Context

The web client now assembles dense views from many REST resources, over-fetching and chaining round trips. Client teams want to shape their own queries.

## Decision

A GraphQL gateway fronts internal services for first-party clients. The underlying REST API remains the contract for third-party integrators.

## Consequences

- Clients fetch exactly the graph they render in one request.
- The gateway needs query-cost limits to stay safe.
- Two API styles now coexist and must be kept consistent. Supersedes ADR-0004.
