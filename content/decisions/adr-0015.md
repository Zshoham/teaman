## Context

Secrets are our most sensitive data. A single static encryption key is both a large blast radius and a rotation nightmare.

## Decision

Each secret is encrypted with a unique data key, which is itself encrypted by a master key held in a managed KMS. Rotating the master key re-wraps data keys without touching ciphertext.

## Consequences

- Master-key rotation is cheap and does not rewrite secret data.
- Compromise of one data key exposes only one record.
- KMS is now on the critical path for every decrypt and must be cached carefully.
